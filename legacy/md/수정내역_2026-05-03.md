## 추가 수정 내역 — 채팅 기반 후속 조치

| 이슈 | 수정 전 | 수정 후 |
|---|---|---|
| FPS가 1.xx대로 하락 | Android 캡처 간격이 `700ms`라 표시 FPS가 1~2fps대로 고정될 수 있었음 | `INTERVAL_MS = 100L`로 복구해 10fps 목표 루프 재적용 |
| 분석 중 표시 고정 | 온디바이스 빈 결과가 서버 fallback 대기나 UI 지연 때문에 `분석 중...` 상태로 남을 수 있었음 | 빈 탐지 결과도 즉시 `장애물 없음`으로 표시하고, 스트림이 멈추면 ImageCapture fallback으로 재시도 |
| 온디바이스 전환 실패 | 서버 URL 기본값 때문에 URL을 비운 것처럼 보여도 실제 분기는 서버 모드로 남을 수 있었음 | 실제 입력된 서버 URL과 기본 URL을 분리하고, 네트워크가 없으면 장애물/찾기 모드는 온디바이스 우선 |
| 시작 직후 "마우스가 사라졌어요" 안내 | 서버 tracker/공간 스냅샷의 오래된 상태가 빈 프레임과 비교되어 사라짐 문장이 나올 수 있었음 | 빈 프레임은 공간 변화 비교에서 제외하고, tracker의 사라짐 변화는 상태 정리만 수행 |
| GPS 대시보드 위치 미표시 | 분석 시작 시 GPS heartbeat가 빠지거나 `0.0,0.0` 좌표가 전송되어 서버 저장이 무시될 수 있었음 | GPS heartbeat 복구, Fused Location/Network/GPS provider 동시 시도, `0.0,0.0`은 앱에서 전송하지 않음 |
| 팀원 위치가 내 위치만 보임 | 서버 `/sessions`, `/team-locations`에 실제 세션이 하나만 들어와 대시보드가 표시할 팀원 데이터가 없었음 | 팀원 앱도 고유 `device_id`로 GPS를 보내도록 위치 획득 경로 보강 및 대시보드 확인 절차 정리 |
| 첫 음성 명령 안내 끊김 | 첫 TTS 안내가 끝나기 전 `startListening()`이 실행되어 `tts.stop()`으로 안내가 끊길 수 있었음 | TTS `onDone` 이후에만 첫 STT 듣기를 시작하도록 변경 |
| feature/myunggwang 병합 | feature 브랜치에 문장 수정이 새로 올라왔고 main 수정 보존 필요 | `feature/myunggwang`를 main에 병합하고, main의 critical 차량 거리/액션 정책은 보존 |
| 서버 모드 FPS 7~8fps | 서버 요청 동시성이 낮고 업로드 이미지가 커서 왕복 시간이 FPS를 제한 | 서버 in-flight를 4로 확대, 업로드 이미지를 480px/JPEG 65%로 축소 |
| 온디바이스 FPS 5~6fps | 카메라 분석 해상도와 ONNX 전처리 메모리 복사가 부담 | 분석 해상도 480x360, ONNX 전처리 배열 복사 축소, 온디바이스 in-flight 및 ORT 스레드 조정 |
