# 2026-04-29 날짜별 정리

## 오늘 목표

- 강사님 피드백에 따라 FPS 10 이상을 목표로 성능을 개선한다.
- GCP 서버 마이그레이션 준비를 진행한다.
- 음성 명령 딜레이와 TTS 무음 문제를 수정한다.
- 디버그 로그와 FPS 표시를 추가한다.

## 사용모델

- YOLO 객체 탐지 모델
- ONNX Runtime Android
- Depth V2
- Android 내장 TTS
- gTTS fallback
- ElevenLabs TTS 옵션

## 구동원리

- Android에서 카메라 프레임을 캡처한 뒤 YOLO/ONNX로 객체를 탐지한다.
- 서버 모드에서는 `/detect` API로 이미지를 보내고 서버가 객체 탐지, 거리 추정, 문장 생성을 처리한다.
- 첫 서버 요청이 느린 문제를 줄이기 위해 서버 시작 시 모델을 워밍업한다.
- TTS는 재생 중 상태를 통합 관리해 문장이 조용히 버려지는 상황을 막는다.
- `VG_PERF`, `VG_DETECT` 로그로 decode, infer, dedup, total 시간을 분리해서 확인한다.

## 목표 실제 구동 유,무

- 대부분 구동 확인.
- NNAPI 하드웨어 가속과 CPU 스레드 병렬 처리를 추가함.
- STT 침묵 감지 시간을 줄여 음성 명령 딜레이를 개선함.
- TTS 무음, 바운딩박스 유지, 분석 중지 버튼 미작동, 첫 감지 지연 문제를 수정함.
- FPS 10 이상은 목표로 설정했으나 기기와 서버 상태에 따라 추가 검증이 필요함.

## 내일 목표

- GCP Cloud Run 서버를 실제 외부 접속 가능한 상태로 배포한다.
- Android 앱에서 서버 실패 시 온디바이스 fallback이 제대로 되는지 확인한다.
- 실제 기기에서 장애물 인식, TTS, bbox 표시를 검증한다.
