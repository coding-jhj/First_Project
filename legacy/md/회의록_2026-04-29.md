## 상세 작업 내역

### 🐛 버그 수정

**TTS 느림 · 분석 실패 · 첫 요청 timeout**
- TTS 느림: ElevenLabs 라우팅 제거 → Android 기본 TTS 전용으로 2~4초 지연 해소
- 분석 실패: `isSpeaking()`이 `ttsBusy`를 항상 체크하도록 수정해 문장 누락 방지
- 첫 요청 timeout: 서버 시작 시 Depth V2 워밍업 추가 (미웜업 시 10~30초 소요, Android 8초 제한 초과)
- HTTP 타임아웃: 연결 5→10초, 읽기 8→20초

**분석 중지 미작동 · 첫 감지 느림 · 바운딩박스 갱신 지연**
- 분석 중지 미작동: `handleSuccess()`에 `isAnalyzing` 체크 추가
- 첫 감지 느림: `startAnalysis()`에 `detectionHistory.clear()` 추가로 이전 세션 잔존 데이터 제거
- 바운딩박스 갱신 지연: `VOTE_MIN_COUNT` 2→1 변경으로 400~800ms 지연 해소

**바운딩박스 유지 버그 · isSending 데드락**
- 바운딩박스 예상보다 오래 유지되던 버그 수정
- `isSending` 플래그 데드락 수정

**NNAPI 재비활성화 (FP16 오류)**
- NNAPI 활성화 시 confidence score가 0으로 수렴 → 탐지 0개 버그
- CPU 4스레드 추론으로 롤백

### ⚡ 성능 개선

| 항목 | 변경 내용 | 기대 효과 |
|---|---|---|
| 프레임 캡처 간격 | 100ms → 50ms | 최대 20fps |
| Depth V2 프레임 스킵 | 매 프레임 → 3프레임마다 1회 (캐싱) | 레이턴시 700ms+ → 300ms 이하 |
| 이미지 업로드 최적화 | file/width/height 동시 반환 | 중복 디코딩 ~20ms 절약 |
| 성능 디버깅 로깅 | CSV 로깅 + 컴포넌트별 타이밍 추가 | 병목 구간 측정 가능 |
