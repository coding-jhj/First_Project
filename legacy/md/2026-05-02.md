# 2026-05-02 날짜별 정리

## 오늘 목표

- Android와 서버 파이프라인을 개선해 FPS 10 이상을 달성한다.
- 서버의 Depth 계산 부담을 줄이고 응답 시간을 낮춘다.
- TTS 안정성과 GPS 위치 갱신을 보강한다.

## 사용모델

- YOLO 객체 탐지 모델
- ONNX Runtime Android
- bbox 기반 거리 추정
- Depth V2 옵션
- FastAPI 서버
- Android TTS/STT

## 구동원리

- 서버에서는 `max_det=8`로 프레임당 후처리 물체 수를 제한한다.
- Depth V2는 기본 OFF로 두고, bbox 기반 거리 추정으로 서버 응답 시간을 줄인다.
- Android는 `INTERVAL_MS`를 줄이고 최대 2개 동시 요청을 허용해 프레임 처리가 서버 응답에 완전히 막히지 않게 한다.
- 최신 응답만 UI와 TTS에 반영해 오래된 응답이 화면을 덮어쓰는 문제를 막는다.
- FPS 표시는 최근 10프레임 이동평균으로 계산한다.

## 목표 실제 구동 유,무

- 구동 확인.
- 서버 응답시간 300ms 이하와 Android 10fps 목표에 맞춘 구조가 반영됨.
- TTS 속도 정상화, tracker 유지 시간 증가, 영어 약어 조사 처리, GPS heartbeat 보강이 반영됨.

## 내일 목표

- 실제 환경에서 FPS, GPS, fallback 동작을 재검증한다.
- Android 표시 상태가 "분석 중"에 고정되는 문제를 확인한다.
- 서버와 온디바이스 모드 전환이 사용자에게 명확히 보이도록 다듬는다.
