# VoiceGuide Q&A Sheet

담당: 문수찬  
목적: 강사님 질문에 팀원이 같은 기준으로 답하게 만든다.

## 역할 질문

| 질문 | 답변 |
|---|---|
| 팀장은 누구인가요? | 정환주가 팀장이고 서버, 프론트엔드, GCP 배포 기준을 맡습니다. |
| Vision/ML 담당은 누구인가요? | 신유득이 모델, 오탐, Depth fallback, 평가 자료를 맡습니다. |
| Android 담당은 누구인가요? | 김재현이 Android/UX와 FPS 개선을 맡습니다. |
| NLG 담당은 누구인가요? | 임명광이 응답 문장과 서버 보조를 맡습니다. |
| Voice/Q&A 담당은 누구인가요? | 문수찬이 STT/TTS 검증과 Q&A 시트를 맡습니다. |

## 서버 질문

| 질문 | 답변 |
|---|---|
| 서버는 무엇을 실행하나요? | `src.api.main:app` 하나를 실행합니다. |
| 배포는 어디에 하나요? | GCP Cloud Run을 기준으로 합니다. |
| 서버가 꺼지면 앱이 멈추나요? | 기본 장애물 탐지는 Android 온디바이스 ONNX로 계속 동작합니다. 서버 기능만 fallback됩니다. |
| legacy 서버는 뭔가요? | `legacy/server_db*`는 과거 DB/실험 코드이고 현재 본 서버가 아닙니다. |
| 대시보드에 아무것도 안 보이면 실패인가요? | 아닙니다. `/dashboard`와 `/status`가 200 OK여도 해당 세션에 탐지 객체, GPS, track 데이터가 없으면 화면이 비어 보일 수 있습니다. |

## MVP 질문

| 질문 | 답변 |
|---|---|
| 발표 MVP는 무엇인가요? | 장애물 안내, 물건찾기, 물건 확인 3개입니다. |
| 왜 3개로 줄였나요? | 기능을 늘리기보다 실제 폰에서 설명하고 검증 가능한 흐름을 안정화하기 위해서입니다. |
| 대시보드/GPS는 MVP인가요? | 서버 연동 확인용 보조 기능입니다. 발표 핵심 MVP는 아닙니다. |
| OCR, 신호등, SOS는 빼나요? | 코드나 아이디어가 있어도 실환경 검증이 부족하면 실험 기능으로 분리합니다. |

## 성능 질문

| 질문 | 답변 |
|---|---|
| FPS가 낮으면 어떻게 개선하나요? | `VG_PERF`로 병목을 측정하고, 직렬 구조를 bounded 병렬 구조로 바꿉니다. |
| interval만 줄이면 되나요? | 아닙니다. 발열, queue 적체, TTS 겹침이 생길 수 있어 추론 시간과 후처리 시간을 같이 봅니다. |
| 목표 FPS는 얼마인가요? | 발표 최소 목표는 5fps 이상이고, 안정적 시연은 8~10fps를 목표로 봅니다. |

## 정확도 질문

| 질문 | 답변 |
|---|---|
| 거리가 정확한가요? | 정확한 측정이 아니라 bbox/Depth 기반 대략 거리 추정입니다. |
| 오탐은 어떻게 줄이나요? | class별 threshold, voting, NMS, 실패 이미지 기록으로 줄입니다. |
| 계단 감지는 완성인가요? | 계단/낙차 후보 감지는 실험 기능으로 설명합니다. |
| 신호등/OCR은 완성인가요? | 발표에서는 실험 기능으로만 말합니다. |

## 음성 질문

| 질문 | 답변 |
|---|---|
| TTS는 서버에서 하나요? | Android 긴급 안내는 OS TTS가 우선입니다. 서버 TTS는 보조/데모 성격입니다. |
| 음성이 겹치면 어떻게 하나요? | Android에서 TTS busy lock, cooldown, alert mode로 겹침을 막습니다. |
| STT가 실패하면요? | 핵심 장애물 안내는 버튼/기본 분석으로 유지하고, 음성 명령은 fallback 문구를 둡니다. |

## 조심할 표현

| 피할 말 | 대신 말할 말 |
|---|---|
| 안전을 보장합니다 | 보행 보조 정보를 제공합니다 |
| 정확한 거리입니다 | 대략적 거리 추정입니다 |
| 모든 상황에서 됩니다 | 현재 테스트 범위에서 동작 확인했습니다 |
| 기능이 다 완성됐습니다 | MVP 기능과 실험 기능을 분리했습니다 |
