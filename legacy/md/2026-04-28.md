# 2026-04-28 날짜별 정리

## 오늘 목표

- 기능 추가보다 현재 동작하는 MVP와 팀 역할을 안정화한다.
- Android 온디바이스 탐지와 서버 연동 구조를 설명 가능하게 만든다.
- 오탐, FPS, TTS 겹침 문제를 우선 수정한다.

## 사용모델

- YOLO11m 기반 객체 탐지
- ONNX Runtime Android
- Depth fallback
- Android 내장 TTS/STT
- PyTorch GPU 환경

## 구동원리

- Android 앱은 서버 없이도 ONNX 모델로 기본 장애물 탐지를 수행한다.
- 서버는 GCP Cloud Run 기준의 `src.api.main:app` 하나로 통일한다.
- 최근 여러 프레임의 탐지 결과를 Voting 방식으로 묶어 오탐과 반복 안내를 줄인다.
- 차량, 칼 같은 위험 물체는 안전 우선으로 즉시 통과시킨다.
- bbox 크기를 거리 대용 지표로 사용해 가까운 물체는 음성, 먼 물체는 비프음으로 안내한다.

## 목표 실제 구동 유,무

- 구동 확인.
- 질문 모드, 실시간 대시보드, GPS 위치 연동, Voting 방식, TTS 겹침 차단이 반영됨.
- 휴대폰 오인식 개선을 위해 휴대폰/노트북 이미지 기반 파인튜닝 파이프라인을 추가함.
- 계단 감지는 강사님 피드백에 따라 한 차례 제거됐다가, 이후 보조 탐지로 다시 검토됨.
