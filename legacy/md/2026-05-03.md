# 2026-05-03 날짜별 정리

## 오늘 목표

- Android FPS, GPS, 온디바이스 fallback 관련 회귀 문제를 수정한다.
- 분석 중 표시 고정, 첫 음성 안내 끊김, GPS 위치 미표시 문제를 해결한다.
- feature 브랜치 병합 후 문장 생성 품질을 유지한다.

## 사용모델

- YOLO 객체 탐지 모델
- ONNX Runtime Android
- bbox 기반 거리 추정
- FastAPI 서버
- Android TTS/STT
- Qwen3-TTS, Azure TTS, gTTS fallback 구조

## 구동원리

- Android 분석 루프에서 빈 탐지 결과도 즉시 "인식 물체 없음"으로 처리해 UI가 멈춰 보이지 않게 한다.
- 서버 URL 기본값과 실제 입력값을 분리해 네트워크가 없을 때 온디바이스 모드가 우선 동작하도록 한다.
- GPS heartbeat를 복구하고 `0.0,0.0` 좌표는 전송하지 않는다.
- 서버 요청 in-flight 수와 업로드 이미지 크기를 조정해 서버 모드 FPS를 높인다.
- TTS 첫 안내가 끝난 뒤 STT를 시작해 첫 문장이 중간에 끊기지 않도록 한다.

## 목표 실제 구동 유,무

- 일부 구동 확인.
- pytest 15개 통과 기록 있음.
- GPS 세션과 팀 위치 확인 결과, 당시 서버에는 특정 Android 세션 1개만 존재함.
- Android assembleDebug는 Gradle transform 캐시 잠금 문제로 실패 기록이 있어 추가 빌드 확인이 필요함.

- 손에 든 물건 인식 같은 실제 사용 명령을 추가한다.
