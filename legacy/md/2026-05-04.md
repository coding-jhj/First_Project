# 2026-05-04 날짜별 정리

## 오늘 목표

- Android 성능과 UX를 개선한다.
- "손에 든 게 뭐야?" 명령을 추가해 바로 앞 물건을 즉시 안내한다.
- TTS 음성과 화면 텍스트가 어긋나는 문제를 줄인다.

## 사용모델

- YOLO 객체 탐지 모델
- ONNX Runtime Android
- bbox 기반 거리 추정
- FastAPI 서버
- Android 내장 TTS
- ElevenLabs TTS 옵션

## 구동원리

- "손에 든 게 뭐야", "이거 뭐야", "바로 앞에 뭐 있어" 같은 STT 키워드를 `들고있는것` 모드로 매핑한다.
- 서버 `routes.py`에서 `mode == "들고있는것"` 분기를 타고 `build_held_sentence()`로 바로 앞 물체 안내 문장을 생성한다.
- bbox 크기와 추정 거리를 이용해 "손에 들고 있는 건 ___예요", "바로 앞에 ___이/가 있어요"처럼 수치 없는 문장을 만든다.
- Android는 메모리 직접 캡처 방식으로 File I/O를 줄이고, ONNX 스레드를 조정해 FPS를 개선한다.
- TTS 시작 콜백 또는 MediaPlayer 시작 직전에 화면 텍스트를 갱신해 음성과 화면을 맞춘다.

## 목표 실제 구동 유,무

- 코드 반영 확인.
- `들고있는것` 모드, STT 키워드, 서버 문장 생성 분기, TTS-text sync 개선이 문서상 반영됨.
- FPS는 8~9에서 10 이상을 목표로 개선했으며 실제 기기별 검증이 필요함.

## 내일 목표

- 실제 Android 기기에서 `들고있는것` 모드와 TTS-text sync를 테스트한다.
- FPS 10 이상이 안정적으로 유지되는지 Logcat `VG_PERF`로 확인한다.
- 최종 발표용으로 MVP 기능, 한계, 시연 순서를 정리한다.
