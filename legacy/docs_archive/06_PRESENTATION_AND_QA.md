# Presentation And Q&A

## 한 줄 소개

> 스마트폰 카메라로 장애물 안내, 물건 찾기, 물건 확인을 한국어 음성으로 지원하는 시각장애인 보행 보조 AI 앱입니다.

## 발표 흐름

1. 문제 정의
   - 기존 서비스는 설명 중심입니다.
   - VoiceGuide는 방향, 거리, 행동 안내를 함께 제공합니다.

2. MVP 소개
   - 장애물 안내
   - 물건 찾기
   - 물건 확인

3. 시스템 구조
   - Android 카메라
   - 온디바이스 ONNX YOLO
   - 서버/GCP 보조 분석
   - 한국어 문장 생성
   - TTS 출력

4. 시연
   - 의자/사람/가방 등 실내 장애물 안내
   - "가방 찾아줘" 같은 찾기 모드
   - "이거 뭐야" 확인 모드

5. 검증과 한계
   - FPS와 추론 시간
   - 오탐/실패 사례
   - 거리 추정의 한계
   - 안전 보조 정보라는 점

## 발표에서 쓰기 좋은 문장

- "정확한 거리를 보장하는 앱이 아니라, 보행 판단에 도움을 주는 보조 안내 앱입니다."
- "서버가 없어도 Android ONNX로 기본 기능은 동작하도록 구성했습니다."
- "기능 수보다 실제 시연 가능한 MVP 안정성을 우선했습니다."
- "오탐이 있는 기능은 무리하게 안전 기능으로 설명하지 않고 실험 기능으로 분리했습니다."

## 예상 질문

**Q. 인터넷이 없어도 되나요?**  
A. 기본 장애물 안내, 찾기, 확인 흐름은 Android 온디바이스 ONNX로 처리하는 방향입니다. 서버는 보조 분석과 대시보드, 저장 기능에 사용합니다.

**Q. 실시간인가요?**  
A. 카메라 프레임을 계속 받지만 모든 프레임을 다 분석하지는 않습니다. 오래된 프레임보다 최신 프레임 중심으로 처리해 실제 안내 지연을 줄이는 구조입니다.

**Q. 거리 정확도는 어느 정도인가요?**  
A. bbox 기반 거리는 대략 추정입니다. 서버에서 Depth를 쓰면 보조적으로 개선할 수 있지만, 발표에서는 "대략 거리"라고 설명합니다.

**Q. 계단도 안전하게 감지하나요?**  
A. 계단/낙차는 위험도가 높아 조심스럽게 설명해야 합니다. YOLO와 Depth 기반 실험은 했지만, 안전 보장을 말하기보다 보조 경고로 설명합니다.

**Q. 경쟁 서비스와 차이는 뭔가요?**  
A. 단순 설명이 아니라 방향, 거리, 행동 안내를 한국어 음성으로 제공하는 점입니다.

## 발표 전 체크리스트

- Android 앱 실행 가능
- 카메라 권한 허용
- TTS 출력 확인
- 서버 `/health` 확인
- 장애물/찾기/확인 모드 각 1회 이상 시연 가능
- 실패 시 설명할 fallback 문장 준비
