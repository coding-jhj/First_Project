## 1. 전체 슬라이드 구성 요약

> 발표 총 20분 = **발표 13~14분** + **데모 5~7분**  
> 1슬라이드 ≈ 1분 기준

| # | 슬라이드 제목 | 핵심 메시지 | 시간 |
|---|---|---|---|
| 1 | 표지 | 프로젝트명, 팀, 한 줄 소개 | 30초 |
| 2 | 문제 정의 | 왜 이 프로젝트인가? | 1분 |
| 3 | 기존 방식의 한계 | 기존 솔루션이 왜 부족한가 | 1분 |
| 4 | 고객여정지도 | 사용자 Pain Point | 1분 |
| 5 | 해결 방향 | 우리의 접근법 | 1분 |
| 6 | 시스템 아키텍처 | 전체 구조 다이어그램 | 1분 30초 |
| 7 | 파이프라인 노드 설명 | 각 단계 입력/출력 | 1분 |
| 8 | 기술 선택 근거 | YOLO11n + YOLO26n fallback + ByteTrack | 1분 30초 |
| 9 | 핵심 AI 성능 지표 | FPS, mAP50, Latency 수치 | 1분 |
| 10 | 구현 및 실험 방법 | 테스트 환경, 비교 실험 | 1분 |
| 11 | 성능 결과 | 표 + 그래프 | 1분 |
| 12 | 진행 경과 | 주차별 타임라인 | 30초 |
| 13 | 데모 안내 | 시연 전 설명 | 30초 |
| 14 | 자체 평가 및 향후 방향 | 잘된 점/부족한 점/개선 방향 | 1분 |

**총 발표: 약 13~14분 → 데모 5~7분 → 합계 20분**

|---|---|

### Slide 5 — 해결 방향

**핵심 메시지:** "우리가 선택한 접근법"

```
3가지 핵심 방향

① 모바일 온디바이스 추론
   → 서버 지연 없이 실시간 처리

② 적응형 모델 전환 구조
   → YOLO11n 기반 + FPS 저하 시 YOLO26n fallback

③ 사용자 중심 피드백
   → 위험도 기반 음성 안내 (수치 → 언어)
```

**핵심 메시지:** "각 노드가 무슨 일을 하는가"

| 노드 | 입력 | 처리 | 출력 |
|---|---|---|---|

### Slide 8 — 기술 선택 근거

**핵심 메시지:** "왜 이 조합인가 — 수치로 설명"

#### 8-1. YOLO11n (기본) + YOLO26n (fallback) 구조

| 항목 | YOLO11n | YOLO26n |
|---|---|---|
| 역할 | 기본 탐지 모델 | FPS 저하 시 fallback |
| 우선순위 | 정확도·안정성 중심 | FPS 안정성 확보 |
| 모바일 추론 | 가능 | 가능 (더 경량) |
| 특징 | 정밀도 높음 | 빠른 추론 속도 |

> 설계 의도: 정상 환경에서는 YOLO11n으로 정확도 확보,  
> 발열·성능 저하 시 자동으로 YOLO26n으로 전환해 FPS 유지

#### 8-2. ByteTrack 선택 근거

| 항목 | SORT | ByteTrack |
|---|---|---|
| ID Switch | 많음 | 적음 |
| 가려짐(occlusion) 처리 | 불가 | 가능 |
| 속도 | 빠름 | 빠름 |

> 선택 이유: 보행 중 일시적으로 가려지는 객체 재추적 필수

#### 8-3. 핵심 문장

> "정확도보다 실시간 사용 가능성을 우선시하여  
> YOLO11n 기반 탐지 + YOLO26n fallback으로 FPS 안정성을 확보했습니다."
