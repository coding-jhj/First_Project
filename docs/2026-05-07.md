## 추가 작업 — yolo11n 모델 통합

### 배경

강사님이 파인튜닝한 TFLite 모델 두 개 수령:
- `yolo11n_320.tflite` — 320×320, 2.8 MB
- `yolo11n.tflite` — 640×640, 2.9 MB

### 모델 출력 포맷 분석

| 파일 | 출력 shape | 비고 |
|---|---|---|
| yolo26n_float32 | `[1, 300, 6]` | NMS 내장, 정리된 결과 |
| yolo11n_320 | `[1, 84, 2100]` | raw 출력 (NMS 없음) |

84 = bbox 4 + COCO 80클래스. 좌표는 정규화(0~1), 클래스 스코어는 sigmoid 적용됨.  
yolo26n도 내부적으로 80클래스 처리 — 이름의 "26"은 클래스 수가 아님.

### 코드 변경 내용 (TfliteYoloDetector.kt)

- `isRawOutput` 플래그 추가: `outputRows == 84` 이면 raw 처리 분기
- `postProcessRaw()` 신규 추가: `[84][N]` 형식 → NMS → `List<Detection>`
- 버그 수정: `4 until 84` 하드코딩 → `4 until rawOutput.size`
- 모델 우선순위: `yolo11n_320 → yolo26n_float32` 자동 폴백

### 결과

- `experiment/yolo11n` 브랜치에서 검증 후 main에 통합
- `backup/main` 브랜치로 이전 상태 보존
- README 업데이트 (ONNX Runtime → TFLite 전환 내용 반영)
