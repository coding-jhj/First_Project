## 음성 안내 시나리오 결정

### 경고 피로 억제

- 음성 안내 빈도를 줄이고 **진동으로 대체**하는 방식 채택
- 갑작스러운 이벤트(예: 우전방 장애물) → 진동으로 대체
  - 예시 문구: "우전방 앞에 뭐가 있어서 조심하세요" → 진동 패턴으로 처리

### 거리 기반 안내 흐름

- 3m 이내 흐름을 보고 **10m → 5m → 3m** 단계별 안내
- 싱크 맞추기가 어렵기 때문에 거리 구간을 나눠 처리

### 네비게이션 모드

- MVP 완성 후 네비게이션 안내 활성화
- 네비게이션 모드에서는 **같은 지점 안내는 1회만** 출력
- 해당 위치에 도달할 때까지 동일 안내 반복하지 않음

## yolo11n 모델 통합

강사님이 파인튜닝한 TFLite 모델 두 개를 수령해 분석 및 main 통합 작업 진행.

### 수정 전 / 수정 후 — 모델 교체

| 항목 | 수정 전 | 수정 후 | 상태 |
|---|---|---|---|
| 기본 추론 모델 | yolo26n_float32.tflite (9.8 MB) | yolo11n_320.tflite (2.8 MB) | ✅ |
| 모델 출력 형식 | `[1, 300, 6]` NMS 내장 | `[1, 84, 2100]` raw 출력 | ✅ |
| 후처리 함수 | postProcessEndToEnd | postProcessRaw (신규 추가) | ✅ |
| 형식 자동 감지 | 없음 | outputRows==84 → raw 분기 | ✅ |
| 클래스 범위 | 하드코딩 84 | rawOutput.size 동적 처리 | ✅ |
| 폴백 모델 | 없음 | yolo26n_float32 자동 폴백 | ✅ |
| yolo11n.tflite (640px) | — | 실험 브랜치에만 유지, main 제외 | 🗑️ |

### 모델 출력 포맷 분석 결과

| 파일 | 출력 shape | 좌표 | 클래스 스코어 |
|---|---|---|---|
| yolo26n_float32 | `[1, 300, 6]` | xyxy (정규화 or 픽셀) | score 직접 출력 |
| yolo11n_320 | `[1, 84, 2100]` | cx/cy/w/h 정규화 0~1 | sigmoid 적용 완료 |

yolo26n 내부 텐서 분석 결과 이름의 "26"은 클래스 수가 아님 — 두 모델 모두 COCO 80클래스.
