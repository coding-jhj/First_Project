# 재현 및 수정 기록 (14:45 수정)

## 요약

MVP 점검 중 `/detect_json` 경로의 SQLite 저장 오류를 재현했고, 서버 MVP 구조와 실제 Android 모델 구현에 맞지 않던 문서/테스트/주석을 함께 정리했다.

## 수정 전후 비교

| 구분 | 수정 전 | 재현/문제 | 수정 후 | 확인 결과 |
|---|---|---|---|---|
| `/detect_json` 최근 탐지 저장 | `detections` 테이블을 이벤트 상세 저장과 최근 프레임 저장이 함께 사용 | SQLite에서 먼저 생성된 `detections(event_id, bbox_x...)` 스키마 때문에 `device_id`, `cx`, `cy` insert 시 `table detections has no column named device_id` 발생 | 최근 프레임 저장 전용 `recent_detections` 테이블 추가. `save_detections()`, `get_recent_detections()`가 새 테이블을 사용하도록 변경 | `/detect_json` API 시뮬레이션 `200 OK`, 최근 탐지 1건 저장 확인 |
| 기존 `/detect` 이벤트 저장 | `detection_events` + `detections`에 원본 이벤트와 객체 행 저장 | `/detect_json` 수정 중 기존 이벤트 저장 경로 회귀 가능성 있음 | 기존 `detections` 테이블은 이벤트 상세 저장용으로 유지 | `save_detection_event()` 시뮬레이션에서 `latest_event evt-1 의자` 확인 |
| 회귀 테스트 | `/detect_json` DB 저장을 직접 검증하는 테스트 없음 | 같은 스키마 충돌이 다시 들어와도 테스트에서 놓칠 수 있음 | `tests/test_api.py`에 `test_detect_json_persists_recent_detections()` 추가 | API 테스트 포함 핵심 테스트 `22 passed` |
| 서버 import 테스트 | `torch`, `ultralytics`를 서버 핵심 import로 요구 | 현재 MVP 서버는 JSON 라우터이고 서버 추론을 하지 않는데, 학습/내보내기 도구 의존성 미설치로 `test_imports.py` 실패 | 서버 런타임 핵심 의존성만 `cv2`, `numpy`, `fastapi`로 정리 | `tests/test_imports.py` 포함 핵심 테스트 통과 |
| README 모델 설명 | `yolo11n.onnx`, `ONNX Runtime`, `YOLOv11n`으로 설명 | 실제 Android 구현은 `yolo26n_float32.tflite`와 TensorFlow Lite 사용 | README를 `yolo26n_float32.tflite`, `TFLite`, `YOLO26n` 기준으로 수정 | README와 실제 파일/코드 명칭 일치 |
| Android 코드 주석 | `MainActivity.kt` 주석에 ONNX Runtime 표현 잔존 | 구현체는 `TfliteYoloDetector`인데 주석이 과거 구조를 가리킴 | 주석을 TFLite 온디바이스 추론 기준으로 수정 | 검색 시 과거 ONNX/yolo11n 문구 제거 |
| Android wrapper | `android/gradlew.bat`만 존재 | Linux/macOS/CI에서 `./gradlew` 실행 불가 | `android/gradlew` 추가 및 실행 권한 부여 | wrapper 호출은 됨. 단 현재 환경에는 Java가 없어 Android 빌드는 `JAVA_HOME is not set`으로 중단 |
