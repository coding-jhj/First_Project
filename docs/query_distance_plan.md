# 거리 안내 — 질문할 때만, 최근 1~3초 요약 방식 설명

> 코드 수정 전 정리본. 무엇을 어떻게 바꿔야 하는지만 적음.

---

## 현재 동작 방식 (이동 중에도 거리 안내함)

지금은 카메라가 켜져 있으면 **항상** 탐지 → 문장 생성 → 거리 포함 TTS 발화.

```
걷는 중 → 의자 탐지 → "왼쪽 약 1.5m에 의자가 있어요."
(이동 중이라 1초 후엔 의자 위치가 바뀌어 있음 → 안내가 틀려짐)
```

**강사님 피드백**: 이동 중에는 사물 위치가 계속 바뀌어서 거리 수치 안내가 물리적으로 맞지 않음.

---

## 바꿀 방향

```
이동 중 (걷는 중)
  → 거리 수치 없이 방향만: "왼쪽에 의자."
  → 위험 물체만 즉시: "차 조심!"

정지 상태 (멈춰서 질문)
  → "앞에 뭐가 있어?" 질문 시
  → 최근 1~3초 탐지 결과 요약: "바로 앞에 의자, 오른쪽에 사람이 있어요."
```

---

## "최근 1초 / 최근 3초 요약"이란?

폰은 50ms마다 카메라 프레임을 처리함. 1초면 약 20프레임.

그 결과를 버퍼에 계속 쌓아두고, 사용자가 질문하면 **버퍼를 꺼내서 요약**하는 것.

```
[버퍼에 쌓이는 것]
0.0초: 의자, 사람
0.1초: 의자
0.2초: 의자, 가방
0.3초: 의자, 사람
     ...
1.0초: 의자

→ 최근 1초 요약: "의자" (거의 매 프레임 등장), "사람" (2번 등장), "가방" (1번 등장)
→ 확정 기준(절반 이상 등장): 의자만 확정 → "바로 앞에 의자가 있어요."
```

**왜 3초짜리도 있나?**  
1초는 너무 짧아서 잠깐 지나간 물체를 놓칠 수 있음.  
3초는 좀 더 넓은 맥락을 봄 — "방금 지나왔던 계단"도 포함됨.  
질문 내용에 따라 쓰는 범위를 다르게 할 수 있음.

## 폰이 추론하는 새 아키텍처에서 어떻게 구현하나

### 필요한 것 두 가지

**① 탐지 결과 버퍼 (폰에서)**

```
현재: detectionHistory → 최근 3프레임(클래스 이름만) 저장
바꿀 것: recentDetections → 최근 N초치 탐지 결과(클래스 + bbox + 시각) 저장
```

타임스탬프와 함께 탐지 결과를 저장해두면 "최근 1초", "최근 3초" 를 언제든 꺼낼 수 있음.

**② 정지 상태 감지 (폰에서)**

폰에는 가속도계 센서가 있음. 이걸 쓰면 "지금 걷고 있나 / 멈춰 있나"를 알 수 있음.

```
가속도 변화가 작다 → 정지 상태 → 거리 안내 허용
가속도 변화가 크다 → 이동 중   → 거리 안내 금지, 방향만
```

또는 더 단순하게: **STT로 질문이 들어왔을 때 = 정지 상태**로 간주.  
(걸어가면서 질문하는 경우는 드물고, 질문 자체가 "멈추고 물어보는" 행위임)

## 서버가 해야 할 일 (새 아키텍처 기준)

현재 서버에 `tracker.py`가 있어서 최근 3초 상태를 `get_current_state(max_age_s=3.0)`로 꺼냄.  
이건 서버가 YOLO 추론을 할 때 만들어지는 구조임.

새 아키텍처에서는:

```
폰 → 탐지 결과 JSON → 서버 전송
서버 → JSON을 DB에 저장 + tracker에 업데이트
사용자가 질문 → 서버에서 tracker 꺼내서 요약 → 폰에 응답
```

즉, 서버 tracker는 **폰에서 보내준 JSON으로 업데이트**하는 구조로 바뀜.  
서버가 이미지를 받아서 직접 YOLO 돌리는 부분만 없어지면 됨.

## 정리 — 바꿔야 할 것 목록

| 위치 | 바꿀 내용 | 이유 |
|---|---|---|
| 폰 (Android) | `detectionHistory` → 타임스탬프 포함 버퍼로 교체 | 1초/3초 요약 가능하게 |
| 폰 (Android) | 이동 중에는 거리 수치 문장 생성 안 함 | 부정확한 안내 방지 |
| 폰 (Android) | 질문 STT → 버퍼 요약 → 거리 포함 문장 생성 | 질문 시에만 거리 안내 |
| 서버 (Python) | tracker를 폰에서 보낸 JSON으로 업데이트 | YOLO 추론 서버 제거 |
| 서버 (Python) | 질문 요청 오면 tracker 꺼내서 응답 | 기존 구조 유지 |

## 질문 시 응답 예시 (목표)

| 상황 | 응답 문장 |
|---|---|
| 의자 1개 | "바로 앞에 의자가 있어요." |
| 의자 + 사람 | "앞에 의자, 오른쪽에 사람이 있어요." |
| 아무것도 없음 | "지금은 주변에 아무것도 없어요." |
| 최근 3초 안에 계단 지나침 | "방금 계단이 있었어요. 주의하세요." |
