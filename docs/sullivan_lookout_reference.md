# 설리번 플러스 & Google Lookout 참고 분석

> 두 앱의 기능을 VoiceGuide에 어떻게 녹일 수 있는지, 저작권은 어떻게 되는지 정리.

---

## 1. 설리번 플러스 (Sullivan Plus) 주요 기능

한국 스타트업 투아트(TUAT)가 SK텔레콤과 협력해 만든 시각장애인 AI 보조 앱.

| 기능 | 설명 |
|---|---|
| AI 자동 인식 | 카메라가 켜지면 자동으로 물체·장면 분석 → 음성 안내 |
| 텍스트 인식 (OCR) | 간판, 서류, 인쇄물의 글자를 읽어줌 |
| 얼굴 인식 | 화면 속 사람의 나이·성별 추정 안내 |
| 이미지 설명 | 전체 장면을 자연어로 설명 |
| 색상 인식 | 화면 중앙 색상 or 전체 화면 주요 색상 안내 |
| 물체 찾기 | 사용자가 목표 물체 선택 → 그 방향으로 음성 유도 |
| 돋보기 | 줌 + 색상 반전 지원 |
| 밝기 감지 | 전면 카메라로 주변 밝기 측정 후 안내 |

**기술 배경**: 10억 장 이상 학습된 A.X 멀티모달 AI 사용. 삼성전자와 협력 중(2024 발표).

---

## 2. Google Lookout 주요 기능

Google이 만든 무료 시각장애인 보조 앱. 온디바이스 추론(TensorFlow Lite) 사용.

| 모드 | 설명 |
|---|---|
| 탐색(Explore) | 카메라를 돌리면 물체·사람·텍스트 위치와 거리 음성 안내 |
| 찾기(Find) | 특정 물체(문, 화장실, 컵 등) 찾아서 방향+거리 안내 |
| 텍스트(Text) | 문서, 간판 텍스트 실시간 읽기 (30개 언어) |
| 문서(Document) | 페이지 전체 글자·손글씨 인식 후 읽어줌 |
| 화폐(Currency) | 지폐 종류와 금액 인식 |
| 식품 라벨(Food Labels) | 식품 포장 바코드·성분·조리법 읽기 |
| 이미지 설명(Images) | 사진 찍고 음성으로 Q&A 가능 |

**기술 배경**: TensorFlow Lite(LiteRT) 온디바이스 추론. 오프라인 상태에서도 OCR 동작 확인됨.

## 4. VoiceGuide에 가져올 수 있는 기능 아이디어

### 설리번 플러스에서 참고할 것

- **색상 인식**: 옷 맞춤 기능(예정 기능)에 활용 가능. YOLO 없이 카메라 픽셀 분석만으로 구현 가능 → 가벼움
- **밝기 감지**: 현재 "어두움" 감지 기능 있음. 설리번처럼 전면 카메라로 측정하는 방식 참고 가능
- **자동 인식 모드(버튼 없이)**: 현재 VoiceGuide가 이미 하고 있음 → 방향 맞음

### Google Lookout에서 참고할 것

- **모드 분리 구조**: 탐색/찾기/텍스트/문서처럼 기능별로 명확히 나눈 UX → STT 키워드 모드 구조와 유사, 더 정교하게 다듬을 수 있음
- **온디바이스 추론**: 강사님 방향과 정확히 일치. Lookout이 TFLite로 구현한 것처럼 VoiceGuide도 ONNX Runtime으로 동일하게 접근 중
- **거리+방향 동시 안내 (Find 모드)**: 찾기 모드에서 "오른쪽 약 2m" 수준의 안내 → VoiceGuide 찾기 모드 개선 시 참고
- **진동(Haptic) 피드백**: 두 앱 모두 사용. 텍스트 인식 성공 시 진동 등 → 청각 외 피드백 추가 아이디어

## 5. 저작권 문제 — 걱정할 필요 없는 이유

### 아이디어는 저작권 보호 대상이 아님

> **"시각장애인에게 카메라로 주변 상황을 음성으로 알려주는 앱"** 이라는 아이디어 자체는 누구도 독점할 수 없음.

저작권이 보호하는 것:
- 소스 코드 그 자체 (똑같이 복사할 때)
- UI 디자인 (화면 레이아웃, 아이콘)
- 텍스트·이미지 등 콘텐츠

저작권이 보호하지 않는 것:
- **기능 아이디어** ("물체 찾기", "거리 안내" 같은 개념)
- **동작 방식의 개념** (보팅, EMA 평활화 같은 알고리즘 아이디어)

### 특허는 별도 문제

- 설리번 플러스나 Lookout이 특정 기술을 **특허 등록**했다면 그 구현 방식은 침해가 될 수 있음
- 단, 특허는 공개 검색이 가능하고, 같은 결과를 **다른 방법으로 구현**하면 문제없음
- 학교 프로젝트·비상업용 목적은 특허 분쟁 실익이 없어서 실질적으로 리스크 거의 없음

### 안전하게 참고하는 방법

| 해도 되는 것 | 하면 안 되는 것 |
|---|---|
| 기능 목록 참고해서 비슷한 기능 직접 구현 | 상대방 앱 소스코드 복사 |
| 동일한 개념을 다른 코드로 구현 | UI 디자인·아이콘 그대로 모방 |
| 공개 API·오픈소스 모델 사용 | 앱 이름·로고 유사하게 사용 |
| 유사한 사용자 경험 제공 | 역공학으로 얻은 알고리즘 그대로 사용 |

## 6. 우선순위 — VoiceGuide에 실제로 추가할 만한 것

프로젝트 기간과 강사님 방향을 고려한 현실적인 우선순위.

| 우선순위 | 기능 | 참고 앱 | 난이도 | 비고 |
|---|---|---|---|---|
| ★★★ | 찾기 모드 거리+방향 개선 | Lookout Find | 낮음 | 기존 코드 수정 |
| ★★★ | 문장 짧게 픽스 | 두 앱 모두 짧음 | 낮음 | SentenceBuilder 수정 |
| ★★ | 색상 인식 | 설리번 플러스 | 낮음 | 픽셀 분석, 모델 불필요 |
| ★★ | OCR (텍스트 읽기) | 두 앱 모두 | 보통 | ML Kit 등 라이브러리 활용 |
| ★ | 화폐 인식 | Lookout | 높음 | 별도 모델 필요 |
| ★ | 얼굴 인식 | 설리번 플러스 | 높음 | 프라이버시 이슈도 있음 |
