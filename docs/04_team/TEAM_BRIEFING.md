# VoiceGuide 팀 브리핑

> 기준일: 2026-05-01  
> 목적: 발표와 Q&A에서 역할, 코드, 현재 MVP 범위를 같은 말로 설명한다.

## 한 문장 소개

VoiceGuide는 카메라로 주변 장애물을 탐지하고, 방향과 대략적 거리를 한국어 음성으로 안내하는 시각장애인 보행 보조 앱입니다.

## 발표 범위

| 구분 | 기능 |
|---|---|
| 발표에서 시연 | Android 카메라 탐지, 방향 안내, 대략적 거리 안내, TTS, 온디바이스 fallback, GCP 대시보드 |
| 실험 기능으로 설명 | OCR, 옷 매칭, SOS, 하차 알림, 신호등, 공간 기억 |
| 말하지 않을 표현 | 정확한 거리 측정, 안전 보장, 100% 계단 감지, 신호등 판단 보장 |

## 예상 질문

### Q. 서버는 어떤 파일을 실행하나요?

`src.api.main:app` 하나입니다. `/detect` 라우터는 `src/api/routes.py`에 있고, `legacy/server_db*`는 본 서버가 아닌 과거 실험 코드입니다.

### Q. GCP 말고 ngrok이나 다른 배포도 쓰나요?

현재 발표와 배포 기준은 GCP Cloud Run입니다. ngrok은 로컬 임시 시연 참고용이고, README에서는 GCP를 주 경로로 설명합니다.

### Q. 서버가 죽으면 앱도 멈추나요?

아닙니다. Android는 ONNX 온디바이스 탐지를 우선 사용하고, 서버 URL이 없거나 요청이 실패하면 온디바이스 fallback으로 기본 장애물 안내를 유지합니다.

### Q. 거리 정보는 정확한가요?

정확한 거리 측정이 아니라 대략적 거리 추정입니다. bbox 면적과 Depth V2 보조값을 사용하고, 발표에서는 "약 n미터", "가까이"처럼 보수적으로 설명합니다.

### Q. 실험 기능은 왜 발표 범위에서 빼나요?

안전 앱에서는 불확실한 기능을 완료 기능처럼 말하면 위험합니다. OCR, SOS, 신호등, 하차 알림 등은 구현 흔적이 있어도 실환경 검증이 부족하면 실험 기능으로 분리합니다.
