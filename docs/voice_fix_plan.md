# 음성 안내 수정 계획 — 쉬운 설명

> 코드 수정 전 정리본. 무엇이 문제이고 어떻게 바꿔야 하는지만 적음.

---

## 현재 음성 안내의 구조 (간단 요약)

```
탐지 결과 → 문장 생성(SentenceBuilder) → 모드 판단 → TTS 발화
                                           ↓
                               critical  : 위험 → 즉시 발화
                               normal    : 일반 → 말 안하고 있을 때만
                               beep      : 멀리 있음 → 소리만
                               silent    : 억제 → 발화 안 함
```

---

## 문제 1 — 실외에서 TTS가 이상하게 나옴

### 원인

현재 코드가 `STREAM_MUSIC`(음악 스트림)으로 TTS를 재생함.
실외에서는 바람 소리, 차 소리 등 배경 소음이 크고, 음악 스트림은 보통 이어폰/스피커 볼륨을 따르기 때문에 **환경에 따라 너무 작게 들리거나 다른 소리에 묻힘.**

### 해결 방향

| 항목 | 현재 | 바꿀 내용 |
|---|---|---|
| 오디오 스트림 | `STREAM_MUSIC` | `STREAM_ACCESSIBILITY` (접근성 스트림) |
| 말하기 속도 | critical: 1.0x, normal: 1.1x | 실외 고려 → critical: 0.95x (약간 천천히), normal: 1.0x |

`STREAM_ACCESSIBILITY`는 Android에서 시각장애인 보조 앱(TalkBack 등)이 쓰는 전용 채널이라, **배경 소음과 섞이지 않고 더 잘 들림.**

## 문제 2 — 음성 시나리오가 픽스되어 있지 않음

"시나리오 픽스"란: **어떤 상황에서 어떤 말을 할지 미리 다 정해두고, 그것만 쓰는 것.**  
지금은 물체 이름, 방향, 거리를 조합해서 즉석으로 문장을 만드는데, 조합 방식에 따라 어색하거나 너무 긴 문장이 나올 수 있음.

### 현재 문장 예시 vs 바꿀 방향

**장애물 일반**

| 상황 | 현재 | 제안 |
|---|---|---|
| 의자가 바로 앞에 | "바로 약 0.5m에, 의자가 있어요." | "바로 앞에 의자." |
| 의자가 왼쪽에 멀리 | "왼쪽 약 3m에, 의자가 있어요." | "왼쪽에 의자." |
| 장애물 없음 | "주변에 장애물이 없어요." | "이상 없음." (짧게) |

**차량 위험**

| 상황 | 현재 | 제안 |
|---|---|---|
| 차가 가까이 | "위험! 바로 앞 자동차! 조심!" | "차 조심!" (더 짧게, 더 빠르게) |

**찾기 모드**

| 상황 | 현재 | 제안 |
|---|---|---|
| 의자 발견 | "의자는 오른쪽 약 1.5m에 있어요." | "의자, 오른쪽." |
| 못 찾음 | "의자가 없어요. 다른 곳을 보여주세요." | 유지 (이건 괜찮음) |

**핵심 원칙**: 실외 시각장애인이 걸어가면서 듣는 거라서 **짧고 명확하게.** 긴 문장은 다 듣기 전에 상황이 바뀜.

## 문제 4 — 보팅과 TTS 타이밍이 꼬임

### 구조 설명

```
프레임 1: 의자 탐지 → 보팅 버퍼에 1표
프레임 2: 의자 탐지 → 보팅 버퍼에 2표 → 통과! → TTS 발화 시도
                                                    ↓
                                           TTS 재생 중이면 → 버려짐 ❌
프레임 3: 의자 탐지 → 3표 → 또 통과 → TTS 재생 중 → 또 버려짐 ❌
```

**TTS가 말하는 동안 새로운 탐지 결과가 다 버려지는 구조.**

사람이 말하는 동안 앞에서 자전거가 갑자기 나타났다면 안내가 안 됨.

### 해결 방향

| 물체 종류 | 현재 | 바꿀 내용 |
|---|---|---|
| 차량, 계단(bypass) | TTS 중에도 즉시 끊고 발화 (이건 잘 됨) | 유지 |
| 일반 물체 | TTS 중이면 버려짐 | TTS 끝난 직후 가장 최근 탐지 결과 1번 발화 |
| 장애물 없음 | 계속 시도 | TTS 중이면 완전히 스킵 (굳이 "없어요"는 안 해도 됨) |

## 정리 — 수정 우선순위

| 순서 | 항목 | 난이도 | 효과 |
|---|---|---|---|
| 1 | 실외 TTS 스트림 변경 (`STREAM_ACCESSIBILITY`) | 쉬움 (한 줄) | 실외 청취 개선 |
| 2 | 문장 짧게 픽스 (SentenceBuilder 수정) | 보통 | 사용성 큰 향상 |
| 3 | TTS 끝난 후 최근 결과 1회 발화 | 보통 | 탐지 누락 방지 |

## 시나리오 픽스 최종안 (확정할 내용)

아래 표를 팀이 보고 "이 말 맞다 / 이건 바꾸자"로 확정하면 SentenceBuilder에 반영.

| 상황 | 발화할 문장 |
|---|---|
| 장애물 없음 | "이상 없음." |
| 일반 물체 (멀리) | "{방향}에 {물체}." |
| 일반 물체 (가까이) | "가까이 {방향}에 {물체}." |
| 가까이 접근 필요 회피 | "가까이 {방향}에 {물체}. {회피 행동}." |
| 차량 위험 | "차 조심!" |
| 동물 위험 | "{동물} 조심!" |
| 계단 감지 | "계단 주의!" |
| 찾기 — 발견 | "{물체} {방향}에 있어요." |
| 찾기 — 못 찾음 | "{물체} 안 보여요. 다른 곳을 보여주세요." |
| 들고 있는 것 | "손에 {물체}." / "바로 앞에 {물체}." |
| 분석 실패 | "잠깐 안 보여요." |
| 분석 중단 (watchdog) | "주의하세요." |
