# 대시보드 24시간 탐지 내역 패널 추가

## 무엇을 바꿨나요?

대시보드 왼쪽 패널을 **위/아래 절반**으로 나눴습니다.

| 영역 | 내용 |
|------|------|
| 위 절반 | 기존 실시간 현황 (위험/주의/안전 + 탐지된 물체) |
| 아래 절반 | **신규** 최근 24시간 탐지 내역 (요약 통계 + 이벤트 목록) |

---

## 왜 이렇게 만들었나요?

### 문제
기존 대시보드는 "지금 이 순간" 탐지 결과만 보여줬습니다.  
"오늘 하루 동안 어떤 물체가 얼마나 탐지됐는지"를 알 수 없었습니다.

### 해결
24시간 누적 내역을 추가해 **장기적인 패턴** 파악이 가능해졌습니다.

---

## 앱 성능(FPS)에 영향이 없는 이유

```
[앱 탐지 흐름]          [대시보드 내역 조회]
앱 → POST /detect       브라우저 → GET /history
     (초당 수회)                  (30초에 1회)
         ↓                            ↓
    DB 저장 (쓰기)            DB 조회 (읽기만)
```

- 호출 주체가 다름: 앱 vs 브라우저
- 호출 빈도가 다름: 초당 수회 vs 30초에 1회
- 작업 종류가 다름: DB 쓰기 vs DB 읽기
- 코드 경로가 다름: 완전히 별개의 엔드포인트

---

## 수정된 파일 3개

### 1. `src/api/db.py`

**추가된 함수: `get_history_24h(session_id, limit=50)`**

```
역할: DB에서 최근 24시간 탐지 이벤트를 조회하고
      위험도별 건수(요약)와 이벤트 목록을 반환.
```

**위험도 분류 기준:**

| risk_score | 분류 |
|-----------|------|
| 0.7 이상 | 위험 (critical) |
| 0.4 ~ 0.7 | 주의 (beep) |
| 0.4 미만 | 안전 (safe) |

이벤트 위험도 = 해당 이벤트 내 물체들 중 가장 높은 위험도

**SQLite / PostgreSQL 모두 지원:**
- PostgreSQL: `NOW() - INTERVAL '24 hours'` 사용
- SQLite: Python으로 24시간 전 시각 계산 후 ISO 문자열 비교

---

### 2. `src/api/routes.py`

**추가된 엔드포인트: `GET /history/{session_id}`**

```
호출자:   브라우저(대시보드)
호출주기: 30초마다 1회
반환:     session_id, period_hours(24), summary, events
```

기존 엔드포인트(/detect, /status 등)와 독립적으로 동작합니다.

---

### 3. `templates/dashboard.html`

**CSS 추가:**
- `.panel-top` / `.panel-bottom`: 패널을 50:50 비율로 분할
- `.history-stats-grid` / `.history-stat-card`: 24시간 요약 카드
- `.history-list` / `.history-item`: 내역 목록 스타일

**HTML 변경:**
- 기존 내용을 `<div class="panel-top">` 으로 감쌈
- `<div class="panel-bottom">` 에 새 내역 섹션 추가

**JS 추가:**
- `HISTORY_POLL_MS = 30000`: 30초 폴링 주기
- `fetchHistory()`: `/history` API 호출
- `renderHistory(data)`: 조회 결과를 화면에 렌더링
- `formatHistoryTime(isoStr)`: ISO 시각 → "HH:MM" 형식 변환
- 세션 변경 시 내역도 즉시 갱신
