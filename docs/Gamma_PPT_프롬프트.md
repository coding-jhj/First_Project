## Slide 2 — 문제 정의

**제목:** 문제 정의

**소제목:** 왜 이 프로젝트가 필요한가?

**왼쪽 — 현황:**
- 국내 시각장애인 약 25만 명 (2024 보건복지부)
- 보행 중 주요 위험: 돌출 장애물, 차량 접근, 낮은 위치 장애물
- 기존 흰 지팡이: 좁은 감지 범위, 원거리 인식 불가

**오른쪽 — VoiceGuide가 해결하는 것:**
1. 실시간 장애물 감지 → 스마트폰 카메라로 전방 객체 탐지
2. 모바일 온디바이스 추론 → 서버 지연 없이 디바이스에서 직접 처리
3. 음성 피드백 안내 → 위험도 기반 직관적 음성 경고

## Slide 3 — 기존 방식의 한계

**제목:** 기존 방식의 한계

**소제목:** 왜 기존 솔루션으로는 부족한가

**표:**

| 방식 | 제공 기능 | 한계 |
|---|---|---|
| 흰 지팡이 | 직접 장애물 탐색 | 좁은 범위, 원거리 인식 불가 |
| GPS 앱 | 위치 기반 경로 안내 | 실시간 장애물 감지 불가 |
| 웨어러블 기기 | 일부 센서 활용 | 고비용, 낮은 보급률 |
| 기존 카메라 앱 | 사진 촬영 안내 | 실시간 피드백 없음 |

**강조 박스:**
> 핵심: 실시간 + 경량 + 모바일 세 조건을 동시에 만족하는 솔루션이 없다

## Slide 5 — 해결 방향

**제목:** 해결 방향

**소제목:** 우리가 선택한 3가지 접근법

**카드 3개 (나란히 배치):**

**카드 1 — ① 모바일 온디바이스 추론**
- 서버 통신 없이 스마트폰에서 직접 추론
- 네트워크 지연 제거, 프라이버시 보호

**카드 2 — ② 적응형 모델 전환 구조**
- YOLO11n 기본 탐지

**카드 3 — ③ 사용자 중심 피드백**
- 위험도 Smoothing → 임계값 초과 시 음성 안내
- 수치(Risk Score)를 자연어로 변환

## Slide 6 — 시스템 아키텍처

**제목:** 시스템 아키텍처

**소제목:** 전체 시스템 구성도

**플로우 다이어그램 (순서대로 노드 배치):**

```
스마트폰 카메라
    ↓
YOLO11n — 기본 객체 탐지
    ↓
FPS 저하 감지
    ↓ (저하 시)
YOLO26n — fallback 전환
    ↓
ByteTrack — 객체 추적
    ↓
위험도 Smoothing 계산
    ↓                    ↓
음성 / 지도 피드백    JSON → 서버 전송
```

**설명 포인트 3개:**
- 클라이언트(모바일): 온디바이스 실시간 처리
- 서버: 로그 저장 및 세션 관리만 담당
- SSOT 구조 — JSON이 단일 데이터 소스

## Slide 7 — 파이프라인 노드 설명

**제목:** 파이프라인 노드 설명

**소제목:** 각 단계의 입력 / 처리 / 출력

**표:**

| 노드 | 입력 | 처리 | 출력 |
|---|---|---|---|
| Camera Input | — | 프레임 캡처 | RGB 이미지 |
| YOLO11n | 이미지 | 객체 탐지 (기본) | BBox, Class, Conf |
| FPS Monitor | 추론 결과 | FPS 측정 + 임계값 판단 | Switch 신호 |
| YOLO26n | 이미지 | 객체 탐지 (fallback) | BBox, Class, Conf |
| ByteTrack | BBox | ID 할당 및 추적 | Track ID, 이동 벡터 |
| Risk Analyzer | Track + Depth | 위험도 Smoothing | Risk Score (0~1) |
| Feedback Module | Risk Score | 임계값 판단 → 음성 생성 | 음성 문자열 |
| JSON Sender | 전체 결과 | 직렬화 | JSON → 서버 |

## Slide 8 — 기술 선택 근거

**제목:** 기술 선택 근거

**소제목:** 왜 이 조합인가 — 수치로 설명

**왼쪽 — YOLO 모델 구조 비교:**

| 항목 | YOLO11n | YOLO26n |
|---|---|---|
| 역할 | 기본 탐지 모델 | FPS 저하 시 fallback |
| 우선순위 | 정확도·안정성 | FPS 안정성 확보 |
| 모바일 추론 | 가능 | 가능 (더 경량) |
| 특징 | 정밀도 높음 | 빠른 추론 속도 |

→ 정상 환경: YOLO11n으로 정확도 확보, 발열·저하 시: YOLO26n으로 FPS 유지

**오른쪽 — ByteTrack 선택 근거:**

| 항목 | SORT | ByteTrack |
|---|---|---|
| ID Switch | 많음 | 적음 |
| 가려짐 처리 | 불가 | 가능 |
| 속도 | 빠름 | 빠름 |

→ 보행 중 일시적으로 가려지는 객체 재추적 필수

**하단 강조 문장:**
> "YOLO11n 기반 탐지를 중심으로, 모바일 안정성을 위해 YOLO26n fallback 구조를 적용했습니다."
