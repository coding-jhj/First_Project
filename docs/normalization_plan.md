### A. 대시보드 — 위치 데이터 문제 해결

**문제:**  
현재 대시보드 지도에는 **실제 GPS 좌표**가 표시됨  
→ 사용자의 실제 이동 경로가 서버에 저장되는 개인정보 문제

**해결 방향:**  
실제 GPS 대신 **시뮬레이터 동선**을 사용

```
실제 GPS 사용 (현재)           시뮬레이터 사용 (목표)
─────────────────────          ──────────────────────
사용자가 실제로 이동            미리 정해진 경로 좌표를
→ 폰 GPS → 서버 전송           코드가 순서대로 전송
→ 지도에 실시간 표시            → 지도에 실시간 표시
(개인정보 위험)                 (개인정보 없음, 재현 가능)
```

**구체적으로:**
- 코드에 좌표 배열을 미리 작성 (예: 학교 복도 → 계단 → 출구)
- 시뮬레이터 실행 시 순서대로 좌표 전송
- 데모마다 동일한 경로 재현 가능 → 발표·포트폴리오에 유리

---

### B. 대시보드 — "물체 없음" 저장 제거

**문제:**  
카메라가 아무것도 감지하지 못한 상태도 DB에 저장되고 있음  
→ 대시보드 24시간 내역에 의미 없는 빈 이벤트가 쌓임

**왜 잘못됐나:**  
- 저장의 목적은 "위험 물체가 감지됐을 때 기록"
- 물체가 없으면 저장할 필요 없음
- 지금은 **저장 이유 없이 저장** 중

**해결:**  
`/detect` 엔드포인트에서 감지된 물체가 0개이면 DB 저장 안 함

```python
# 수정 전
save_to_db(detections)  # 빈 리스트도 저장됨

# 수정 후
if detections:          # 물체가 있을 때만 저장
    save_to_db(detections)
```

### C. DB — 저장 근거 검토

**문제:**  
서버 DB에 어떤 데이터가 왜 저장되는지 근거가 불명확함

**점검 기준:**
1. 이 데이터가 없으면 기능이 안 되는가?
2. 이 데이터를 나중에 꺼내서 쓰는 코드가 있는가?
3. 대시보드·사용자에게 어떤 가치를 주는가?

**저장해야 하는 것:**
- 탐지된 물체가 있는 이벤트 (위험도 판단에 사용)
- 세션 정보 (사용자 연결 관리)

**저장 불필요한 것:**
- 물체 없는 이벤트 (→ 위에서 제거)
- 중복 저장 (같은 물체가 연속으로 탐지돼도 매번 저장 중인 경우)

---

### D. 대시보드 — 데모용 시나리오 확정

**문제:**  
대시보드가 뭘 보여주는 화면인지 설명이 어려움

**목표:**  
발표자가 3문장으로 설명 가능한 데모 흐름 만들기

**제안 시나리오:**

```
1. 시뮬레이터 실행
   → 미리 짜인 경로(복도 이동)를 따라 좌표 전송 시작

2. Android 앱에서 실시간 감지
   → 사람/장애물 발견 시 음성 안내 출력
   → 탐지 결과를 서버로 전송

3. 대시보드에서 확인
   → 지도: 이동 경로 실시간 표시
   → 왼쪽 패널: 지금 감지된 물체 + 위험 수준
   → 아래 패널: 오늘 하루 탐지 이력
```

### E. GCP 서버 — VS Code 연결

**설정 방법:**  
`.vscode/sftp.json` 또는 Remote-SSH config 파일로 GCP 서버에 직접 연결 가능

```
VS Code에서 GCP 서버 파일 직접 수정 가능
→ 서버에 SSH 접속 → 코드 편집 → 저장 즉시 반영
→ Cloud Run 재배포 없이 빠른 테스트 가능
```
