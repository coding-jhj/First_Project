## 미팅 발언 템플릿 (강사님 요구 형식)

> **"샘플링/전처리해 모델 학습 검증결과 — 데이터 규격 및 모델 구조 이상없고, 학습은 현재 어느 정도이며, 지표는 어느 정도 나올 것으로 예상됩니다."**

VoiceGuide 팀 답변:

> "계단 감지 파인튜닝은 DuckDuckGo로 8개 쿼리 × 50~80장 = 최대 480개 URL 수집 후 다운로드 성공 **404장**을 확보했습니다. train/val 85:15 분리, pseudo-label로 전처리했으며 데이터 규격(640×640 YOLO COCO+1클래스 81개)과 모델 구조 이상 없습니다. 학습은 2단계(백본 freeze 10ep + 전체 50ep) 완료, **mAP50=0.992** 달성했습니다. 온디바이스 오탐률이 높아 Depth 맵 12구역 분석으로 대체 중이며, 추가 실내 파인튜닝(`indoor_nav_v1`) 파이프라인을 준비했습니다."

## 실험/연구 노트 (강사님 필수 기록 항목)

### [A] 계단 파인튜닝 — 완료

| 항목 | 내용 |
|------|------|
| **데이터 수집** | DuckDuckGo 이미지 검색, 8개 쿼리 (indoor staircase 80장, hallway stairs 60장 등) |
| **총 수집 이미지** | 404장 (URL 최대 480개 → 다운 성공분) |
| **Train/Val 분리** | Train 344장 / Val 60장 (15% val ratio) |
| **전처리** | pseudo-label — YOLO11m 자동 라벨 + 계단 bbox 하단 중앙 고정 (cx=0.5, cy=0.7, w=0.9, h=0.6) |
| **증강** | ultralytics `augment=True` (flip, mosaic, scale, HSV 등 기본 파이프라인) |
| **베이스 모델** | YOLO11m (COCO80 사전학습) |
| **파인튜닝 전략** | 2단계 — ①백본 freeze 10ep (lr=1e-3) → ②전체 50ep (lr=1e-4, cosine, label_smoothing=0.1) |
| **클래스 수** | 81 (COCO80 + stairs) |
| **이미지 크기** | 640×640 |
| **배치** | 12 (RTX 5060 8GB 기준) |
| **결과** | **mAP50=0.992** |
| **적용 결정** | 온디바이스 오탐률 높아 → 서버 연결 시 Depth 맵 12구역 분석으로 대체 |

---

### [B] Depth Anything V2 거리 추정 — 운영 중

| 항목 | 내용 |
|------|------|
| **모델** | Depth Anything V2 Small (vits, 24.8M params) |
| **입력** | 640px 리사이즈 JPEG (앱에서 75% 압축 후 전송) |
| **출력** | 역깊이 맵 → 5단계 거리 (0.5m / 1m / 2m / 3.5m / 5m+) |
| **보수 추정** | bbox 내 픽셀 하위 30% 사용 (가장 가까운 점 기준) |
| **계단/낙차 감지** | 바닥 12구역 깊이 분산 분석 |
| **Fallback** | Depth 모델 없을 시 → bbox 면적 비율로 거리 추정 |
| **검증 현황** | `/health` 엔드포인트로 로드 상태 확인 완료 |
| **미완료** | 실환경 거리 임계값 정량 측정 (4/28 기준 진행 중) |
