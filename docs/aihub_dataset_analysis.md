## 데이터셋 1: 가구·가전·사무기기 사용 모션 캡처 데이터

- **AI Hub ID:** 71485
- **링크:** https://aihub.or.kr/aihubdata/data/view.do?dataSetSn=71485

### 데이터셋 개요

실생활에서 일반적으로 사용되는 60종의 가구·가전·사무기기에 대해, 다양한 체형/연령/성별 인원이 사용하는 동작을 촬영한 **모션 캡처 중심** 데이터셋.

| 항목 | 내용 |
|---|---|
| 모션캡처 건수 | 67,509건 |
| 영상 | 202,527건 |
| 이미지 | 337,545장 |
| 총 녹화 시간 | 약 300시간 |
| 포맷 | BVH (모션), MP4 (영상), JPG (이미지), FBX (3D모델) |
| 관절 데이터 | 전신 77개 관절 3D 좌표 |
| 객체 수 | 60가지 (가구 45%, 가전 55%) |

### VoiceGuide 관련성 평가

**관련성: 낮음 (⚠️ 직접 활용 어려움)**

이 데이터셋의 핵심 목적은 **사람의 신체 동작(포즈) 데이터**를 수집하는 것이다. 이미지와 영상이 포함되어 있지만, 라벨이 객체 바운딩박스(bbox)가 아닌 **관절 좌표 기반 모션 데이터**로 구성된다.

VoiceGuide가 필요한 것은 "실내 장애물(사물)을 탐지하는 YOLO bbox 학습 데이터"인데, 이 데이터셋은 그 용도로 설계되지 않았다.

**쓸 수 있는 부분이 있다면:**
- 이미지(JPG 337,545장)에는 실내 환경 + 가구/가전이 등장하므로, 이 이미지에서 **별도로 bbox 라벨링**을 추가한다면 YOLO 학습 데이터로 전환 가능
- 단, 그 작업량이 매우 크고 원래 라벨 포맷(관절 좌표)과 완전히 별도 작업이 필요해 비효율적

**결론: 이 데이터셋은 활용하지 않는 것이 맞다.**

## 데이터셋 2: 가상 실내 공간 3D 합성 데이터

- **AI Hub ID:** 71765
- **링크:** https://aihub.or.kr/aihubdata/data/view.do?dataSetSn=71765

### 데이터셋 개요

실제와 동일하게 구축된 가상 실내 공간의 3D 모델 + 2D 렌더링 이미지 + 라벨링 데이터. 주거/상업/교육 등 다양한 실내 공간 유형을 포함한 합성(Synthetic) 데이터셋.

| 항목 | 내용 |
|---|---|
| 3D 공간 모델 | 200개 |
| 3D 에셋 | 3,000개 |
| 2D 렌더링 이미지 | 28,000장 (7가지 조명/렌더링 조건) |
| 라벨링 | Panoptic Segmentation (픽셀 단위) + 3D Cuboid |
| 포맷 | FBX (3D), JPG (이미지), JSON (라벨) |

### 실내 객체 분류

| 대분류 | 예시 객체 | 비중 |
|---|---|---|
| 일반가구 | 소파, 침대, 책장, 의자, 테이블 | 37% |
| 욕실가구 | 세면대, 양변기, 욕조 | 12% |
| 인테리어 | 액자, 파티션, 쿠션 | 16% |
| 주방가구 | 식탁, 쿡탑, 수전 | 10% |
| 일반가전 | TV, 냉장고, 에어컨 | 8% |
| 조명 | 천정/벽/바닥 조명 | 8% |
| 기타 | 커튼, 문, 창문 | 9% |

### VoiceGuide 관련성 평가

**관련성: 중간 (⚡ 조건부 활용 가능)**

VoiceGuide가 탐지해야 할 실내 장애물(소파, 의자, 테이블, 침대, 문 등)이 이 데이터셋에 다수 포함되어 있고, 28,000장의 렌더링 이미지는 훈련 데이터로 쓰기에 규모가 충분하다.

### 활용 가능한 시나리오

#### 시나리오 A: 실내 특화 클래스 학습 데이터 보강

현재 VoiceGuide는 COCO 80클래스 중 일부만 사용하도록 축소 예정이다. COCO에는 실내 가구(소파, 의자, 침대 등)가 포함되어 있지만, **한국형 실내 환경**(장판, 좌식 테이블, 붙박이장 등)에 특화된 이미지는 부족하다. 이 데이터셋의 이미지를 보조 학습 데이터로 사용하면 실내 환경 탐지 정확도를 높일 수 있다.

**전환 작업 필요:**
```
JSON (Panoptic Segmentation) → YOLO bbox 포맷 (.txt) 변환
```
Panoptic segmentation mask에서 각 객체의 bbox를 추출하는 변환 스크립트가 필요하다. 기술적으로는 어렵지 않다.

```python
# 변환 예시 개념 (mask → bbox)
from pycocotools.coco import COCO
# segmentation mask의 bounding box를 계산
# x_min, y_min, x_max, y_max → YOLO 정규화 좌표로 변환
```

#### 시나리오 B: 조명 변화에 강인한 모델 학습

7가지 조명 조건(1,500K~8,000K, 밝기 0~4,000+)으로 렌더링된 이미지가 있어서, **어두운 실내 환경**에서의 탐지 강인성을 높이는 데 활용할 수 있다.

VoiceGuide 현행 기능 중 "어두움 감지"가 있는데, 이 데이터의 저조도 렌더링 이미지는 해당 시나리오의 훈련 데이터로 직접 활용 가능하다.

### 핵심 제약: 합성 데이터 도메인 갭

이 데이터셋은 **가상(Synthetic)** 데이터다. 실제 스마트폰 카메라로 촬영한 이미지와 시각적 특성이 다르다.

| 합성 데이터 특성 | 실제 카메라 특성 |
|---|---|
| 완벽한 조명/렌더링 | 노이즈, 블러, 왜곡 |
| 정확한 기하학적 배치 | 임의의 촬영 각도 |
| 완벽한 텍스처 | 반사, 그림자 변화 |

따라서 이 데이터만으로 학습하면 실제 환경에서 성능이 떨어질 수 있다. **실제 촬영 이미지와 혼합해서 사용**하거나, **pre-training용 보조 데이터**로만 쓰는 것이 적합하다.

### VoiceGuide에서 실용적인 데이터 전략

지금 당장 가장 빠른 방법은 **직접 촬영 + 라벨링**이다. 팀원들이 실제 실내 공간(복도, 거실, 사무실 등)을 스마트폰으로 촬영하고, [Roboflow](https://roboflow.com) 같은 툴로 bbox를 직접 라벨링하면 도메인 갭 없이 고품질 데이터를 얻을 수 있다.

AI Hub 71765 데이터셋은 그 **보조 수단**으로, 다음 순서로 사용하는 것을 권장한다:

1. 실제 촬영 데이터로 우선 학습
2. 학습 데이터가 부족한 클래스(예: 침대, 욕실 기기)에 한해 71765 데이터 보충
3. 저조도 시나리오 강인성 테스트에 71765 조명 변화 이미지 활용
