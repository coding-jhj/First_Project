# 참고 PDF 벤치마킹 PPT 제작 지침

참고 파일: `3.기초생활수급자_지원_음성_모델.pdf`

이 문서는 참고 PDF의 발표 흐름을 VoiceGuide에 맞게 벤치마킹하기 위한 지침입니다.  
목표는 참고 자료의 구조를 그대로 베끼는 것이 아니라, **문제 규명 → 핵심 알고리즘 → 케이스 시연 → 발전 과제** 흐름을 VoiceGuide에 맞게 재구성하는 것입니다.

---

## 1. 참고 PDF에서 가져올 장점

참고 PDF는 아래 흐름이 좋습니다.

1. 프로젝트 개요
2. 가설 정의
3. 데이터 분석으로 문제 정의
4. 핵심 사용자군 분류
5. 솔루션 제안
6. 시스템 처리 프로세스
7. 핵심 매칭 알고리즘
8. 사용자 Case 시연
9. 향후 발전 과제
10. 마무리 메시지

VoiceGuide도 이 흐름을 적용하면 발표가 더 설득력 있어집니다.

다만 VoiceGuide PPT는 제출 조건도 반드시 만족해야 합니다.

| 제출 필수 목차 | VoiceGuide 반영 방식 |
|---|---|
| 프로젝트 개요 | 문제 정의, 목표, 주요 기능, 핵심 가치 |
| 프로젝트 팀 구성 및 역할 | 3팀 역할표 |
| 프로젝트 수행 절차 및 방법 | 시스템 처리 프로세스, 아키텍처, 파이프라인, 핵심 매칭 알고리즘 |
| 프로젝트 수행 경과 | 구현 결과, 검증 결과, 데모 영상 흐름, 시행착오 |
| 자체 평가 의견 | 잘된 점, 부족한 점, 향후 개선 방향 |

| PPT 내 포함 제출 목록 | VoiceGuide 반영 방식 |
|---|---|
| 파이프라인 구성 캡처 및 노드 설명 | Android 파이프라인 슬라이드 |
| 전체 파이프라인 구조도(아키텍처) | Android/Server 분리 아키텍처 슬라이드 |
| 고객여정지도(pain point 정리) | 고객여정지도와 Pain Point 슬라이드 |
| 데모 영상(5~10분 이내) | 검증 및 데모 슬라이드에 시연영상 흐름 명시 |

## 2. VoiceGuide에 맞게 바꿀 핵심 방향

참고 PDF의 핵심 매칭 알고리즘은 아래 구조입니다.

```text
Age + Status + History
→ 사용자 상황 분류
→ 맞춤형 시나리오 제공
```

VoiceGuide에서는 이렇게 바꿉니다.

```text
Object Class + Direction + Distance + Risk
→ 현재 위험 상황 분류
→ 음성·진동 안내 시나리오 제공
```

즉, 우리 PPT의 핵심 알고리즘은 “복지 대상자 매칭”이 아니라 **보행 위험 안내 매칭 알고리즘**입니다.

---

## 3. VoiceGuide 핵심 매칭 알고리즘

### 알고리즘 이름

PPT에서는 아래 이름 중 하나를 사용하면 좋습니다.

- 위험 상황 매칭 알고리즘
- 객체-위험도 안내 매칭 알고리즘
- 보행 위험 피드백 매칭 알고리즘
- Detection-to-Feedback Matching

추천 이름:

> 객체-위험도 안내 매칭 알고리즘

### 입력값

| 입력 | 의미 | 예시 |
|---|---|---|
| Object Class | 감지된 객체 종류 | chair, person, door, stairs |
| Direction | 객체 방향 | 정면, 왼쪽, 오른쪽 |
| Distance | 거리 추정 | bbox 기반 약 1.5m |
| Confidence | 탐지 신뢰도 | 0.91 |
| Risk Score | 위험도 점수 | 0.35 / 0.55 / 0.75 |

### 처리 과정

```text
CameraX 프레임
→ TFLite YOLO 탐지
→ bbox/class/confidence 추출
→ Vote/Dedup으로 순간 오탐 완화
→ IoU Tracking으로 같은 객체 연결
→ EMA Smoothing으로 흔들림 완화
→ distance/risk 계산
→ 안내 시나리오 매칭
→ TTS·진동·UI 출력
```

### 출력값

| 출력 | 예시 |
|---|---|
| TTS 문장 | 정면 약 1.5m에 의자가 있어요. |
| 진동 패턴 | SHORT / DOUBLE / URGENT |
| UI 표시 | bbox, class, confidence |
| 서버 기록 | POST /detect JSON |
| 대시보드 표시 | 현재 객체, GPS, 최근 탐지 이력 |

## 4. VoiceGuide 매칭 테이블 예시

참고 PDF의 `Age / Status / History / Scenario` 표처럼, VoiceGuide는 아래 표를 사용합니다.

| 객체/상황 | 방향 | 거리/위험도 | 안내 시나리오 | 피드백 |
|---|---|---|---|---|
| 의자, 문, 표지판 | 정면 | 가까움 / SHORT | 전방 장애물 안내 | 짧은 진동 + TTS |
| 사람, 자전거 | 좌/우 | 중간 / DOUBLE | 이동 객체 주의 | 두 번 진동 + TTS |
| 차량, 오토바이 | 전방/측면 | 높음 / URGENT | 즉시 주의 안내 | 긴급 진동 + 강한 TTS |

발표 문장:

> VoiceGuide는 탐지 결과를 그대로 말하지 않고, 객체 종류와 방향, 거리, 위험도를 조합해 사용자에게 맞는 안내 시나리오로 매칭합니다.

### 10. 핵심 매칭 알고리즘 상세

참고 PDF가 age/history/status를 나눠 설명한 것처럼, VoiceGuide도 2~3장으로 나눌 수 있습니다.

#### 9-1. Object Class

내용:

- 객체 종류별 위험도 가중치
- 차량/오토바이 등 위험 객체는 높은 우선순위
- 의자/문/계단은 보행 장애물로 안내

#### 9-2. Direction & Distance

내용:

- bbox 위치로 방향 추정
- bbox 크기 기반 거리 추정
- 정면·가까움일수록 높은 위험도

주의:

- Depth 모델로 정확한 거리 계산 완료라고 쓰지 말 것
- ByteTrack 정식 통합처럼 쓰지 말 것

- 긴 문단 최소화
- 실제 YOLO 실험 화면과 대시보드 캡처 활용
- 색상은 사용자 지정 테마 우선

### 각 슬라이드 화면 구성

| 슬라이드 유형 | 구성 |
|---|---|
| 가설/Key Insight | 큰 문장 + 근거 2개 |
| 데이터 분석 | 큰 숫자 + 짧은 설명 |
| 프로세스 | 5단계 흐름도 |
| 핵심 알고리즘 | 매칭 표 |
| Case 시연 | 사용자 발화 + AI 안내 + 화면 캡처 |
| 발전 과제 | 3단계 로드맵 |

## 7. 핵심 매칭 알고리즘을 PPT에서 잘 보이게 하는 법

가장 중요한 슬라이드는 `핵심 매칭 알고리즘`입니다.

이 슬라이드는 아래 구조로 만듭니다.

```text
탐지 결과
Object / Direction / Distance / Risk
        ↓
매칭 규칙
위험도 + 사용자 모드
        ↓
출력
TTS / 진동 / UI / 서버 기록
```

표와 흐름도를 같이 쓰면 좋습니다.

왼쪽:

- 입력값 4개 카드

가운데:

- Matching Engine

오른쪽:

- 출력값 4개 카드

하단:

- 예시 1줄

```text
chair + 정면 + 가까움 + SHORT → "정면 약 1.5m에 의자가 있어요." + 짧은 진동
```

## 8. 주의할 점

참고 PDF처럼 “알고리즘”이라는 말을 쓰려면, 단순 기능 나열이 아니라 입력과 출력이 명확해야 합니다.

반드시 보여줄 것:

- 입력값: 객체, 방향, 거리, 위험도, 모드
- 처리: 안정화, 위험도 계산, 매칭 규칙
- 출력값: TTS, 진동, UI, 서버 기록
- 케이스: 실제 사용자 상황 2~3개

피할 것:

- 서버가 YOLO 추론한다고 표현
- Depth 모델로 정확한 거리 계산 완료 표현
- 미측정 FPS/mAP50 수치 작성
- “완전한 안전 보장” 표현

## 9. 최종 PPT에 반드시 들어갈 기능 정리 내용

기능 정리 md 파일 내용은 PPT에서 빠지면 안 됩니다. 다만 그대로 길게 넣지 말고 아래처럼 압축합니다.

| 기능 정리 내용 | PPT 반영 방식 |
|---|---|
| 실제 동작 기능 | 프로젝트 개요, 구현 결과 슬라이드에 3~5개 키워드로 압축 |
| Android/Server 역할 | 아키텍처 슬라이드에 좌우 분리 구조로 표시 |
| YOLO 모델 입력/출력 | 파이프라인 노드 설명에 포함 |
| TTS·진동 안내 | 핵심 매칭 알고리즘 출력값과 시연 Case에 포함 |
| 검증 결과 | 수행 경과 슬라이드에 확보/미측정 분리 |
| 시행착오 | 자체 평가 의견에 잘된 점/부족한 점/개선 방향으로 정리 |
