# VoiceGuide 기능 정리

## 핵심 정의

VoiceGuide는 스마트폰 카메라로 **지금 앞의 위험**을 감지하고, Android에서 음성·진동으로 즉시 알려주는 시각장애인 보행 보조 MVP입니다.

길 안내 앱처럼 목적지까지의 경로를 안내하는 것이 아니라, 사용자가 이동 중 바로 앞의 객체와 위험을 더 빨리 인지하도록 돕는 안전 보조 레이어입니다.

## 실제 동작 기능

| 영역 | 기능 | 현재 설명 방식 |
|---|---|---|
| 카메라 | CameraX 기반 실시간 프레임 입력 | Android 앱에서 카메라 프레임을 받아 분석 |
| 객체 탐지 | TFLite YOLO 온디바이스 추론 | 서버가 아니라 Android에서 실행 |
| 화면 표시 | bbox, class, confidence 표시 | 탐지 결과를 카메라 화면 위에 표시 |
| 안정화 | Vote/Dedup, IoU Tracking, EMA Smoothing | 순간 오탐과 중복 알림 완화 |
| 위험 판단 | 거리, 중심성, 클래스 기반 risk score | 모든 객체를 똑같이 알리지 않음 |
| 진동 안내 | 위험도별 진동 패턴 | SHORT, DOUBLE, URGENT |
| 음성 안내 | 한국어 TTS 안내 | `정면 약 1.5m에 의자가 있어요.` |
| 서버 연동 | `POST /detect`, `POST /gps` | 탐지 JSON과 위치 정보 업로드 |
| 대시보드 | 현재 객체, GPS, 이력 확인 | 서버 기록을 지도/목록으로 확인 |

## 개선 필요 기능

아래 항목은 완료된 성능처럼 쓰지 말고, 추가 측정 또는 개선 방향으로 분리합니다.

| 항목 | 현재 표현 |
|---|---|
| Android 실제 기기 FPS | 추가 실기 측정 필요 |
| 모델별 mAP50 | 추가 정량 검증 필요 |
| 메모리 사용량 | 장시간 실행 안정성 확인 필요 |
| TTS-UI latency | 사용자 체감 지연 측정 필요 |
| 저조도/야간 성능 | 실제 보행 환경 검증 필요 |
| 거리 추정 정확도 | bbox 기반 추정의 한계, 거리 보정 필요 |
| yolo26n 자동 전환 | fallback 후보, 자동 전환 검증 완료로 표현 금지 |
| 실제 사용자 테스트 | 향후 UAT 필요 |

## 발표에서 강조할 점

- 현재 MVP는 “실제 동작 흐름”을 보여주는 데 집중합니다.
- Android가 즉시 판단하고 서버는 기록과 대시보드를 담당합니다.
- 없는 수치나 완료되지 않은 모델 성능을 과장하지 않습니다.
- 시행착오와 개선 방향을 숨기지 않고, 프로젝트 완성도를 높이는 근거로 보여줍니다.
