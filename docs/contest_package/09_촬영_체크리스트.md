# VoiceGuide 촬영 체크리스트

## 촬영 목표

- [ ] 최종 영상 길이 5~10분 기준으로 촬영
- [ ] 권장 편집 길이 7분 30초 안팎으로 준비
- [ ] 5분 압축본과 10분 확장본 모두 만들 수 있게 실제 시연 클립을 넉넉히 확보
- [ ] 설명 이미지보다 실제 앱 화면, bbox, 서버 로그, 대시보드 화면을 우선 사용

## 촬영 전 준비

- [ ] 앱이 설치된 Android 기기 준비
- [ ] 카메라 권한, 마이크 권한, 위치 권한 확인
- [ ] 서버 URL 설정 확인
- [ ] 서버 실행 확인
- [ ] 대시보드 접속 확인
- [ ] 화면 녹화 기능 확인
- [ ] 조용한 환경에서 TTS 소리 확인
- [ ] 의자, 문, 계단, 사람 등 테스트 객체 준비

## 장면별 촬영 분량

| 구간 | 권장 편집 시간 | 촬영해야 할 원본 |
|---|---:|---|
| 오프닝 | 0:00~0:30 | 앱 아이콘/첫 화면, 대표 bbox 화면 |
| 문제 정의 | 0:30~1:20 | 보행 상황 이미지/아이콘, 짧은 보조 화면 |
| 서비스 개념 | 1:20~2:00 | 길 안내와 전방 위험 보조를 비교할 화면 |
| 전체 구조 | 2:00~2:50 | Android, Server, Dashboard 흐름도 캡처 |
| 객체 탐지 | 2:50~4:20 | 객체별 bbox 탐지 클립 여러 개 |
| 음성·진동 | 4:20~5:10 | TTS 출력, 진동 패턴 설명 화면 |
| 서버 업로드 | 5:10~6:00 | API 요청/로그/성공 화면 |
| 대시보드 매칭 | 6:00~6:50 | 같은 session_id가 대시보드에 반영되는 화면 |
| 모델/검증 | 6:50~7:30 | 모델 파일, 테스트 결과, 추가 측정 항목 |
| 시행착오 | 7:30~8:20 | 문제/배운 점/개선 방향 표 |
| 마무리 | 8:20~8:50 | VoiceGuide 제목, 핵심 문장 |

## Android 앱 녹화

아래 장면을 각각 20~40초씩 2~3번 녹화합니다. 최종 편집에서는 좋은 부분만 잘라 5~10분 구성에 맞춥니다.

- [ ] 앱 실행 첫 화면
- [ ] 카메라 진입
- [ ] 의자 bbox 탐지
- [ ] 문 bbox 탐지
- [ ] 사람 bbox 탐지
- [ ] 계단 또는 위험 객체 bbox 탐지
- [ ] TTS 안내 장면
- [ ] 진동 패턴을 자막/아이콘으로 보여줄 장면
- [ ] 서버 업로드가 발생하는 장면
- [ ] 같은 세션이 대시보드에 반영되는 장면
- [ ] 객체 탐지 실패 또는 흔들림이 개선되는 장면이 있으면 별도 확보

## 서버/대시보드 녹화

- [ ] `POST /detect` 업로드가 발생하는 로그
- [ ] `POST /gps` 업로드가 발생하는 로그
- [ ] 대시보드에서 현재 객체 확인
- [ ] 대시보드에서 GPS 위치 확인
- [ ] 대시보드에서 탐지 이력 확인
- [ ] 필요 시 `/health` 또는 `/status/{session_id}` 확인 화면
- [ ] 같은 session_id로 앱과 대시보드가 연결되는 장면
