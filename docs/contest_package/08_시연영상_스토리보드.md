# VoiceGuide 시연영상 스토리보드

## 목표

PillyPilly 참고 영상처럼 휴대폰 목업과 큰 문구를 사용하되, VoiceGuide는 실제 기술 동작이 보이도록 구성합니다.

권장 출력:

- 1920 x 1080
- 30fps
- 5분 ~ 10분
- 권장 길이: 7분 30초 안팎
- MP4

## 전체 흐름

| 구간 | 시간 | 장면 | 핵심 메시지 |
|---|---:|---|---|
| 1 | 0:00~0:30 | 오프닝 | VoiceGuide는 지금 앞의 위험을 알려주는 앱 |
| 2 | 0:30~1:20 | 문제 정의 | 길 안내만으로는 부족함 |
| 3 | 1:20~2:00 | 서비스 개념 | 보행 인프라를 대체하지 않는 보조 안전 레이어 |
| 4 | 2:00~2:50 | 전체 구조 | Android 즉시 판단, 서버 기록 |
| 5 | 2:50~4:20 | 객체 탐지 | TFLite YOLO 온디바이스 추론 |
| 6 | 4:20~5:10 | 음성·진동 | 위험도 기반 즉시 피드백 |
| 7 | 5:10~6:00 | 서버 업로드 | POST /detect, POST /gps 기록 |
| 8 | 6:00~6:50 | 대시보드 매칭 | 클라이언트와 대시보드 세션 연결 |
| 9 | 6:50~7:30 | 모델/검증 | 확보 수치와 추가 측정 분리 |
| 10 | 7:30~8:20 | 시행착오 | 개선 근거로 정리 |
| 11 | 8:20~8:50 | 마무리 | 보조 안전 레이어 |

> 5분에 맞춰 줄여야 할 경우 5~8번 시연 구간을 각각 30~40초로 압축합니다.
> 10분에 가깝게 늘려야 할 경우 5번 객체 탐지와 8번 대시보드 매칭 구간에 실제 녹화 화면을 더 넣습니다.

## 장면별 구성

### 1. 오프닝

화면:

- 큰 제목 `VoiceGuide`
- 팀명 `3팀`
- 부제 `실시간 객체 탐지 기반 시각장애인 보행 보조 앱`
- 스마트폰 목업 안에 YOLO bbox 화면

자막:

```text
스마트폰 카메라로 지금 앞의 위험을 감지합니다.
```

### 2. 문제 정의

화면:

- 보행 상황 아이콘 3개
- 짧은 큰 문장

자막:

```text
길을 아는 것만으로는 부족합니다.
중요한 것은 지금 앞의 위험입니다.
```

### 3. 서비스 개념

화면:

- 길 안내 앱과 VoiceGuide의 역할 차이
- `경로 안내`가 아니라 `현재 전방 위험 보조`
- 보조 안전 레이어라는 문구

자막:

```text
VoiceGuide는 길 안내를 대체하지 않습니다.
지금 앞의 위험을 한 번 더 알려주는 보조 안전 레이어입니다.
```

### 4. 전체 구조

화면:

- Android 흐름을 크게
- Server 흐름은 보조로

자막:

```text
판단은 Android에서 즉시,
서버는 기록과 대시보드를 담당합니다.
```

### 5. 객체 탐지

화면:

- 실제 앱 녹화 화면
- bbox가 가장 잘 보이는 장면
- 의자, 사람, 문, 계단 등 테스트 객체별 짧은 클립

자막:

```text
01. 실시간 객체 탐지
CameraX 프레임을 TFLite YOLO로 분석합니다.
```

### 6. 음성·진동 안내

화면:

- TTS 문장 바
- 진동 아이콘/파형
- SHORT / DOUBLE / URGENT

자막:

```text
02. 음성·진동 안내
위험도에 따라 안내 방식이 달라집니다.
```

### 7. 서버 업로드

화면:

- Android 화면
- `POST /detect`
- `POST /gps`
- 서버 로그 또는 API 성공 표시

자막:

```text
03. 서버 업로드
탐지 결과와 위치를 서버에 기록합니다.
```

### 8. 대시보드 매칭

화면:

- Android → `POST /detect`, `POST /gps` → Dashboard
- 같은 session_id가 대시보드에 표시되는 장면

자막:

```text
04. 클라이언트-대시보드 매칭
앱에서 보낸 탐지 결과가 대시보드에 반영됩니다.
```

### 9. 모델과 검증

화면:

- 모델 카드
- 검증 수치 카드
- 추가 측정 항목

자막:

```text
05. 모델 설명과 검증
확보된 수치와 미측정 항목을 분리했습니다.
```

### 10. 시행착오

화면:

- 문제 / 배운 점 / 개선 방향 3열

자막:

```text
06. 시행착오
실패 기록이 아니라 개선 근거입니다.
```

### 11. 마무리

화면:

- VoiceGuide
- 3팀
- 스마트폰 + 대시보드 미니 그래픽

자막:

```text
지금 앞의 위험을 한 번 더 알려주는 보조 안전 레이어
```
