# TTS·진동 안내 정리

## 목적

VoiceGuide의 사용자 피드백은 화면을 보지 않아도 현재 위험을 이해하도록 돕는 것입니다.

실제 시연 기준은 아래 흐름으로 정리합니다.

```text
객체 탐지 → 위험도 판단 → 한국어 TTS → 진동 패턴 → 서버 기록
```

## TTS 안내

탐지 결과는 사용자가 바로 이해할 수 있는 짧은 한국어 문장으로 변환합니다.

예시:

```text
정면 약 1.5m에 의자가 있어요.
오른쪽 앞에 장애물이 있습니다.
앞에 사람이 있습니다.
```

## 진동 안내

위험도에 따라 진동 패턴을 다르게 표현합니다.

| 위험도 | 의미 | 피드백 |
|---|---|---|
| SHORT | 낮은 주의 | 짧은 1회 진동 |
| DOUBLE | 중간 주의 | 짧은 2회 진동 |
| URGENT | 즉시 주의 | 긴급 진동 |

## 발표에서 강조할 점

- 서버 응답을 기다리지 않고 Android에서 사용자 안내를 먼저 처리합니다.
- 모든 객체를 똑같이 알리지 않고 위험도에 따라 안내 강도를 조절합니다.
