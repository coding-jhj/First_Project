# 거리 추정 방법들 — 쉬운 설명

> depth(깊이 AI 모델)를 빼기로 했으니, 대신 쓸 수 있는 가벼운 방법들 정리.

---

## 먼저 — "거리 추정"이 왜 필요한가

YOLO는 화면에서 물체를 찾아서 박스를 그려주는데, **몇 미터 앞에 있는지는 알려주지 않음.**  
"의자가 바로 코앞에 있다 vs 10미터 저 멀리 있다"를 구분하려면 거리 추정이 필요함.

depth AI 모델이 그걸 해줬는데 **너무 무거워서 FPS가 안 나오는 게 문제.**

---

## 방법 1 — 박스 크기로 가늠 (현재 사용 중)

**아이디어**: 물체가 가까울수록 화면에서 박스가 크게 잡힘. 박스 크기로 거리를 역산.

```
박스가 크다 → 가깝다
박스가 작다 → 멀다
```

- 지금 이미 이 방식으로 "가깝다 / 중간 / 멀다" 3단계 구분하고 있음
- AI 모델 추가 없음, 연산 거의 없음
- **문제점**: 원래 크기가 다른 물체끼리 비교가 어려움
  (트럭은 원래 크게 잡히고, 칫솔은 원래 작게 잡히니까)

---

## 방법 2 — 물체 실제 크기를 미리 알고 계산 ← 추천

**아이디어**: 사람 어깨 너비는 평균 45cm, 자동차 폭은 약 1.8m처럼 **물체별 실제 크기를 미리 알면** 화면에서 얼마나 크게 잡혔는지를 보고 거리를 계산할 수 있음.

쉽게 비유하면:

> 멀리 있는 사람은 작게 보이고, 가까이 있는 사람은 크게 보임.  
> 사람 키가 170cm라는 걸 알고 있으면, 화면에서 몇 픽셀로 보이는지만 측정해도 거리가 나옴.

```
거리 = (실제 크기 × 카메라 초점거리) / 화면에서의 픽셀 크기
```

### 물체별 기준 크기 테이블 (미리 정해두는 것)

| 물체 | 기준 | 평균 크기 |
|---|---|---|
| 사람 | 어깨 너비 | 0.45m |
| 자동차 | 차 폭 | 1.8m |
| 버스 | 차 폭 | 2.5m |
| 오토바이 | 폭 | 0.8m |
| 의자 | 좌석 너비 | 0.5m |
| 문 | 문 높이 | 2.0m |

- AI 모델 추가 없음, FPS 영향 없음
- "약 1.5미터 앞에 의자" 같은 수치 안내 가능
- **문제점**: 어린이, 작은 개 등 크기 편차가 큰 경우 오차 있음

---

## 방법 3 — 박스 위치(발 위치)로 거리 추정

**아이디어**: 카메라를 가슴 높이에 고정하면, 멀리 있는 물체일수록 화면 위쪽에 보임. 가까운 물체는 화면 아래쪽에 박스 하단이 찍힘.

```
화면

─────────────────  ← 지평선 (먼 곳)
     [  의자  ]    ← 멀리 있음 (박스 하단이 위쪽)


  [    의자    ]   ← 가까이 있음 (박스 하단이 아래쪽)
─────────────────  ← 화면 바닥
```

- AI 모델 추가 없음, 연산 거의 없음
- 시각장애인 앱은 카메라 위치가 어느 정도 일정해서 쓸 만함
- **문제점**: 카메라가 위아래로 기울면 오차 큼. 날아다니는 물체(새 등)는 적용 불가

---

## 방법 4 — 경량 depth AI (참고만)

depth AI를 가벼운 버전으로 바꾸는 방법도 있긴 한데,  
강사님이 "depth는 빼라"고 하셨으니 일단 패스.

---

## 결론 — 이 프로젝트에서 어떻게 쓸 것인가

미팅 방향: **"거리는 질문할 때만 알려줌. 이동 중에는 거리 수치 안내 안 함."**

| 상황 | 쓸 방법 |
|---|---|
| 이동 중 장애물 감지 | 방법 1 (박스 크기 3단계) — 지금처럼 "가깝다 / 멀다"만 구분 |
| "앞에 뭐 있어?" 질문 시 | 방법 2 (실물 크기 테이블) — "약 1.5m 앞에 의자" 수치 안내 |

AI 모델 추가 없이, 물체별 실제 크기 표 하나만 추가하면 됨.
